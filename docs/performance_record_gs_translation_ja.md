# performance_record.gs の処理内容（日本語整理）

本資料は `sheet_scripts/performance_record.gs` の主要処理を、仕様確認しやすい形に要約したものです。

## 1. メニューと入口
- スプレッドシート起動時に以下の3メニューを追加する。
  - リンク変換（D列などのリンクを `HYPERLINK()` へ統一）
  - エクスポート（更新日を条件にExcel出力）
  - 歌唱DB整理（重複整理のDRY RUN / 本実行）

## 2. リンク変換
- `convertUiLinksToHyperlink()` は対象列（既定D列）を走査し、
  - すでに `=HYPERLINK(...)` のセルはスキップ
  - RichText / 数式 / 文字列からURLを抽出
  - 抽出できたセルを `=HYPERLINK("URL", "ラベル")` に変換
- `DRY_RUN=true` の場合は書き換えずログのみ。

## 3. 更新日自動挿入
- `onEdit(e)` で、対象シートの A〜F列が編集された行について G列へ当日を記入。
- G列自体を編集したときは再帰防止のため処理しない。

## 4. 更新日ベースのExcel出力
- `getUniqueDates()` で G列の日付をユニーク化し新しい順で返す。
- `exportSinceDates(selectedDateStrs)` で最小選択日以降を抽出して新規スプレッドシートを作成し、
  - 1行目にヘッダー
  - 2行目以降に本文
  - C列はRichText本文のみ
  - D列は表示文言先頭8桁を `yyyy/mm/dd` 化
  - 出力名は `歌唱DByyyymmdd.xlsx`

## 5. 歌唱DB整理（cleanupSongRecords）
同一キーは「A列アーティスト + B列曲名」で判定。

### Step 1: 同一URL重複削除
- 同一キー内で D列URL が完全一致する行を重複とみなす。
- 比較順は「古い順」：
  1) G列更新日が古い方
  2) 同値なら上の行（下の行を新しい扱い）
- 先頭1件を残し、それ以外を削除対象にする（実行時は実削除、DRY RUN時は候補表示のみ）。

### Step 2: 同一曲内で1件だけ残す
- Step 1後に同一キーで複数残る場合、次の優先で keeper を1件決定。
  1) C列種別優先：歌ってみた > 歌枠 > ショート > その他
  2) 同順位なら D列表示文言先頭8桁(yyyymmdd) が新しい方
  3) さらに同値なら G列更新日が新しい方
  4) さらに同値なら上の行を優先
- keeper 以外は削除対象。

## 6. 現状で未実装の点
- 削除対象を `archive` シートへ移動する機能はない（削除のみ）。
- `archive` シート自動作成・ヘッダー整備機能はない。
